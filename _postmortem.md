## D. 方法论沉淀（标注验证等级）

### 1. 无头 Chrome 渲染 SVG→PNG 出图管线 ⭐⭐
**核心**：本地没有 cairosvg/magick/inkscape 时，用已装的 Chrome 无头模式把"内联 SVG 的 HTML"截成高清 PNG，且能加载 Web 字体（系统字体兜底）。
**来源**：3 版封面 + 4 张宣发卡 + 2 张游戏截图，全程复用同一管线。
**验证状态**：⭐⭐ 本次跨 9+ 次出图稳定成立（同环境、单项目；跨项目/跨月待验）。
**操作规则**：
1. SVG **内联进 HTML**（外链 SVG 不应用网页字体）；字体走 `<link>` + 系统衬线兜底。
2. `chrome --headless=new --force-device-scale-factor=2 --window-size=W,H --virtual-time-budget=8000 --screenshot=out.png file:///render.html`。
3. `--virtual-time-budget` 等字体/网络就绪；`device-scale-factor=2` 出 2× 高清。
4. 要"真机截图"效果：渲染页注入 auto-play 脚本（可读同页 `let` 全局变量点正确按钮），手机视口（≤600px）触发移动布局。
**边界**：依赖本机装了 Chrome；字体被墙时回退系统字体（非阻塞加载可避免卡白屏）。

### 2. 浏览器插件自动化的能力边界 ⭐⭐
**核心**：Chrome 插件能驱动建项目/填表/勾选，但有三类硬边界——**登录/验证码/最终发布**（须人本人）、**文件上传沙箱**（只能传会话已连接目录，无人值守模式下目录授权弹窗不可用 → 用户手动选文件）、**安全域名拦截**（如 mp.weixin.qq.com 直接被拦）。
**来源**：itch 发布全过程 + 微信后台访问被拦。
**验证状态**：⭐⭐（itch + 微信两处独立印证；且与平台通则一致）。
**操作规则**：先判断"哪些步骤插件能做、哪些必须人来"，把人工步骤打包成清晰清单交给用户，不要假装能代劳。

### 3. 并行子代理批量生成 + 脚本校验组装 ⚠️
**核心**：大批量结构化内容（200 桩双语案件），用多个子代理按互斥分类分批写 **JSON**，再用脚本统一**校验（schema/配比/查重）+ 组装（json.dumps 转 JS 字面量，零转义 bug）+ 编译校验（node --check）+ 真机抽测**。
**来源**：案件库 16→200。
**验证状态**：⚠️ 首次（单次成立，跨任务待验）。
**操作规则**：让子代理只产 JSON（最易校验）；组装用 `json.dumps(ensure_ascii=False)` 直接产合法 JS 字符串；分类互斥以避免跨批重复；最后必过"语法校验 + 浏览器实测"双关。

### 4. CJK 假斜体的语言感知修复 ⚠️（但有坚实排版常识支撑）
**核心**：中文没有真斜体，浏览器对 CJK 合成的倾斜很丑；用 `html[lang="zh"] …{ font-style: normal }` 让**拉丁保留斜体、中文转正体**，而非全局砍斜体。
**来源**：标题"点下去"、判语金句等。
**操作规则**：依赖页面 `documentElement.lang`；强调改用颜色/字重代替斜体做强调；顺带：CJK 正文行高给到 ~1.8、小标签字距别用过宽的 .2em。

### 5. H5 → 微信小程序移植要点 + 合规红线 ⚠️
**核心**：小程序无 DOM，渲染层须重写为 WXML/WXSS + `setData`，但**纯数据/逻辑可原样复用**（从 script.js 抽 `let CASES` 前的数据段 + `module.exports` 即成 CommonJS 模块）。**合规红线**：微信"游戏"须做小游戏（需企业主体），小程序一般禁游戏类目 → 反诈科普可走"教育/工具"类目但有被驳风险。
**来源**：wechat-mp 移植。
**验证状态**：⚠️ 首次；合规规则属平台事实。

### 6. Astro public/ 托管静态游戏 + 数据驱动入口 ⚠️
**核心**：把整个静态游戏丢进 Astro 站 `public/<slug>/` 即原样部署到 `/<slug>/`；首页入口做成**数据驱动卡片**（加游戏=加一条 `games.js`），链接用 `import.meta.env.BASE_URL` 兼容多 base。
**来源**：tiaozhuxiansheng.com/fraud-desk + 首页"可玩"区。
**验证状态**：⚠️ 首次。

### 7. Git Bash 把 `BASE_PATH=/` 转成 `/D:/Git` ⭐⭐
**核心**：MSYS 路径转换会把命令行里的 `/` 误转成 Windows 路径，导致 Astro base 错乱、CSS 路径变 `/D:/Git/_astro/...`。
**验证状态**：⭐⭐（本次复现 + above-the-web 部署记忆中已有同款记录）。
**操作规则**：传含 `/` 的环境变量给构建时，用 PowerShell（`$env:BASE_PATH='/'`）或 `MSYS_NO_PATHCONV=1`。CI 在 Linux 上无此问题。
